Security information

foauth.org grants access to dozens of social networks, so security is a very serious concern. The entire source code of the site is available on GitHub and the rest of the software stack is listed on the right-hand side of this page, including what versions are currently in use. View the privacy information for details of what we store and what we do with it.

If you've found any security issues in the current code or operation of the site, please report it to security@foauth.org so it can be dealt with as quickly as possible.

A note about password resets

Rather than conventional password resets over email, foauth.org allows you to use your existing authorizations to temporarily verify your identify. This is easier to implement on our end, and faster to reset on your end. Because most services offer to remember your login information indefinitely, it's possible that anyone with physical access to your computer could access any account you're currently logged into.

That's an unfortunate situation on its own, but it gets worth when you consider that our password reset procedure means an attacker could use this process to gain control of all your authorized services. To combat this, a password reset will immediately remove all of your authorization tokens form foauth.og. You'll still see which ones you had previously authorized, but everything sensitive will have been purged from the system. This also makes password reset a sort of panic button, if you ever need to wipe out your authorizations.
